# Hermes Resource Adapter (MemPalace MCP)

MemPalace MCP currently exposes **tools only** (`tools/list`, `tools/call`) and does not implement `resources/list` or `resources/read`.

If Hermes expects MCP resources, use `hermes-mempalace-adapter.js` as a protocol shim.

If you want a ready-to-wire Hermes-friendly wrapper with common entrypoints,
use `hermes-mempalace-wrapper.js`.

## What it fixes

- Translates `resources/list` into taxonomy-driven synthetic resources.
- Translates `resources/read` into MemPalace tool calls.
- Accepts legacy room paths like `wing_user/room_preferences`.
- Preserves standard tool flow (`tools/call` passes through untouched).

## Supported resource URIs

- `wing_user/room_preferences` (legacy shorthand)
- `mempalace://room?wing=wing_user&room=room_preferences`
- `mempalace://search?wing=wing_user&room=room_preferences&q=profile&limit=5`
- `mempalace://status`
- `mempalace://taxonomy`
- `mempalace://kg/stats`

## Mapping behavior

- `resources/list` -> `mempalace_get_taxonomy`
- `resources/read` for room/search URIs -> `mempalace_search`
- `resources/read` for `mempalace://status` -> `mempalace_status`
- `resources/read` for `mempalace://taxonomy` -> `mempalace_get_taxonomy`
- `resources/read` for `mempalace://kg/stats` -> `mempalace_kg_stats`

## Integration example

```js
import { createHermesMemPalaceAdapter } from './hermes-mempalace-adapter.js';

const adapter = createHermesMemPalaceAdapter({
  callRpc: async (request) => {
    // Existing Hermes transport to MemPalace MCP process/socket.
    return sendToMcpServer(request);
  },
  defaultSearchQuery: 'recent important memories decisions learnings preferences',
  defaultSearchLimit: 8,
});

// Use this as the single request handler Hermes talks to.
export async function handleMcp(request) {
  return adapter.handleRequest(request);
}
```

## Hermes wrapper example (drop-in)

```js
import { createHermesMemPalaceWrapper } from './hermes-mempalace-wrapper.js';

const hermesMcp = createHermesMemPalaceWrapper({
  sendJsonRpc: async (request) => {
    // Your existing Hermes MCP transport call:
    return sendToMcpServer(request);
  },
  defaultSearchQuery: 'recent important memories decisions learnings preferences',
  defaultSearchLimit: 8,
});

// Use whichever entrypoint your Hermes runtime expects:
export const handleRequest = hermesMcp.handleRequest;
// aliases available:
// hermesMcp.onRequest(...)
// hermesMcp.request(...)
// hermesMcp.readResource('wing_user/room_preferences')
// hermesMcp.listResources()
```

## Notes

- This adapter intentionally returns room resources as JSON payloads generated by search.
- MemPalace does not support room-name slashes in wing/room identifiers, so `wing/room` path format must always be exactly two segments.
